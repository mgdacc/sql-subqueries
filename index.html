Subqueries
1 / 12

¿Qué es una Subquery?

Una subquery (o subconsulta) es una sentencia `SELECT` que se encuentra anidada dentro de otra sentencia SQL principal.

Es como hacer una pregunta para obtener un dato que necesitas para poder hacer tu pregunta final.

2 / 12

Subqueries en la Cláusula `WHERE`

Es el uso más común. Permite filtrar los datos de la consulta principal basándose en el resultado de la subconsulta.

-- Dame los empleados que ganan MÁS que el salario promedio.
SELECT Nombre, Salario
FROM Empleados
WHERE Salario > (SELECT AVG(Salario) FROM Empleados);
3 / 12

`WHERE` con Operadores `IN` y `NOT IN`

La subconsulta puede devolver una lista de valores que se usan para filtrar.

-- Muéstrame los clientes que SÍ han realizado un pedido.
SELECT Nombre
FROM Clientes
WHERE ClienteID IN (SELECT ClienteID FROM Pedidos);
4 / 12

Subconsultas Correlacionadas

Una subconsulta correlacionada es aquella que depende de la consulta externa para cada fila que procesa. Es como un bucle.

-- Para CADA cliente de la consulta externa,
-- la interna busca si tiene pedidos.
SELECT Nombre FROM Clientes C
WHERE EXISTS (
  SELECT 1 FROM Pedidos P WHERE P.ClienteID = C.ClienteID
);
5 / 12

`EXISTS`: ¿Existe o no?

`EXISTS` es un operador que devuelve `TRUE` si la subconsulta produce al menos una fila, y `FALSE` si no. Es muy eficiente para comprobaciones de existencia.

A menudo es más rápido que `IN` en grandes volúmenes de datos.

Subqueries en la Cláusula `SELECT`

Se usan para añadir una columna calculada a cada fila del resultado. La subconsulta debe devolver un único valor (una fila, una columna).

-- Para cada cliente, calcula su número total de pedidos.
SELECT
  Nombre,
  (SELECT COUNT(*) FROM Pedidos P 
   WHERE P.ClienteID = C.ClienteID) AS TotalPedidos
FROM Clientes AS C;
7 / 12

Subqueries en la Cláusula `FROM`

Permite tratar el resultado de una consulta como si fuera una tabla temporal. A esto se le llama "Tabla Derivada".

-- Calcula el promedio de las ventas totales por categoría.
SELECT AVG(TotalVentas) AS PromedioDeVentas
FROM (
  SELECT Categoria, SUM(Monto) AS TotalVentas
  FROM Pedidos
  GROUP BY Categoria
) AS VentasPorCategoria; -- ¡El alias es obligatorio!

El Problema de la Legibilidad

Las subconsultas anidadas pueden volverse muy difíciles de leer, depurar y mantener.

SELECT * FROM A WHERE A.id IN
(SELECT B.id FROM B WHERE B.id IN
(SELECT C.id FROM C WHERE ...))

¿Existe una forma mejor?

9 / 12

CTEs: La Alternativa Moderna

Las `Common Table Expressions` (CTEs), usando la cláusula `WITH`, permiten nombrar una subconsulta para usarla después. Mejoran enormemente la legibilidad.

WITH VentasPorCategoria AS (
  SELECT Categoria, SUM(Monto) AS TotalVentas
  FROM Pedidos
  GROUP BY Categoria
)
SELECT AVG(TotalVentas)
FROM VentasPorCategoria;
10 / 12

Resumen de Subqueries

En `WHERE`: Para filtrar datos dinámicamente.

En `SELECT`: Para crear columnas calculadas.

En `FROM`: Para usar un resultado como tabla temporal.

`CTEs (WITH)`: La forma moderna y legible de organizar subconsultas complejas.
